# Sign in to a registry

Just like with `docker login`, the `policy` CLI requires you to sign in to an OCIv2-compliant registry.

- To sign in to the `opcr.io` registry, you'll need to [create an OPCR account](/docs/command-line-interface/policy-cli/open-policy-registry). After setting up an account, use the GitHub account you registered with, and a GitHub personal access token (PAT) as your password.

  :::note
  You can create a GitHub PAT on this [page](https://github.com/settings/tokens).
  :::

  ```bash
  policy login -u <GitHub-account> -p <GitHub-PAT>
  ```

- To sign in to the GitHub Container Registry or other registries, use `policy login -s <server-name>`. For example, to login to GHCR:

  ```bash
  policy login -s ghcr.io -u <GitHub-account> -p <GitHub-PAT>
  ```

## Options

`-u <GitHub-account>`: specify the GitHub account as the OPCR user

`-p <GitHub-PAT>`: specify a GitHub personal access token (PAT) as the OPCR password

`-s <registry-hostname>`: specify the registry hostname

`--password-stdin`: read the password from `stdin`

## Example

```bash
echo $GITHUB_PAT | policy login -u ogazitt --password-stdin
```
